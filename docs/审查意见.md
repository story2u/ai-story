## 一、整体项目质量评估

这是一套**极严肃、极自洽的"以文档为状态源"多 Agent 创作系统**。它真正的水平不在"能不能写小说",而在**把信息隔离、maker-checker、回退预算做成了可审计的工程纪律**,而且这些纪律在 ch-0001 的真实生产中**确实生效并拦下了真问题**。

| 维度 | 评分 | 一句话 |
|---|---|---|
| 系统 / Harness 设计 | **9/10** | 三公理 + 五层循环宪法 + RBAC,严密到接近"创作操作系统" |
| 闸门有效性(实证) | **8.5/10** | 不是纸面规则:隔离泄漏、连续性错字、地名硬伤都被真实拦下 |
| 世界观 / 力量体系 | **8/10** | `power-system.md` 可机械判定,"规矩 vs 道理"嵌进了设定层 |

| 流程落地(本章实跑) | **7/10** | 链路真异模型,但 stand-in 侵蚀了多模型收益 |

### 最强的地方

**1. "信息隔离靠构造"不是口号,在真实 prompt 里成立。** 实际派给沈砚的演员 prompt(`relay/ch-0001/sc-0001-01-b01-shen-yan.r1/prompt.md`)全文只有"角色卡 + YOU ARE + 认知包 + 公开场记 + 本拍指令"五块,**零仓库路径、零他人 INNER**,盲区段(`:73-74`)只写"你不知道那位柯先生的来历"——只提示存在、不泄露内容。正是 `HARNESS.md §1①` 承诺的"不递那张牌"。

**2. 闸门真的在干活(最有说服力的证据)。** 单模型一遍过不可能发现的问题,被异模型 checker 抓住:
- **隔离泄漏被拦**:`HANDOFF.md:68` 记录 P2 认知包"DeepSeek stand-in r1 因沈砚包泄露柯九真相作废"——记忆员把反派底牌写进主角的包,当场作废重做。
- **跨文档连续性被拦**:`relay/ch-0001/ch-0001-prose.r2/revision-brief.md` 抓到正文 r1 三处:给镇差私自起名"赵四"、旧砚"贴身/桌角"位置链断裂、"七日"复核窗口应为"三日"。
- **地名硬伤被拦**:验状"青槐镇"≠登记名,总编 r1 打回,六处事实源统一为"碑亭镇"。

**3. 力量体系可裁定,不是文青式空话。** `power-system.md` 给了练气十境 / 武夫九境的 order 表、压制规则(高 1 显著压制、高 2 碾压、常规不可越 2 阶杀敌)、越级"合法出口 + 前置"对照表。总编据此能机械判"战力崩坏"——同类设定文档罕有做到。

### 真正的薄弱处

**1. 通道可靠性正在侵蚀多模型的核心收益——这是 v1 的死因以轻症形式复发。** v2 立项就是为根治"一切角色坍缩成单一 stand-in 模型"。但接力板显示:沈砚 B2.1 因 Codex 额度耗尽改 DeepSeek stand-in、柯九 B1.5 因 Gemini 容量失败改 DeepSeek stand-in、认知包 P2 也走 DeepSeek。结果 `stand_in:4`(发布前自检 `I-W01` 指出只有 1 次有作者批准),且**柯九一章之内被三个模型演过**(Gemini b03 / DeepSeek b05 / Antigravity sc-02)。`reviews/ch-0001.pre-publish.r2.md` 的 `I-W02` 还诚实记下:b05 的 DeepSeek stand-in 产物被 DeepSeek 的 L0 抽检覆盖(同模型自检)。系统对此**高度自知并如实记账**是优点;但问题在复发,是头号系统风险。

**2. 吞吐量是能不能写完这本书的现实拷问。** 单章 3215 字耗费 `dispatch:58 / retry:9 / 作废:7`(`HANDOFF.md:58`)。arc-01 目标 80,000 字 ≈ 25 章,按此节奏是 ~1200+ 次主要靠人肉 CLI 的调度。对"验证流程"足够,对"完成长篇"是严峻协调成本。`remake-v2-plan.md` 自己也承认这轮是为**证明流程**而非冲量——该定位要保持清醒。

**3. 文档陈旧 / 溯源缺口。**
- `README.md:34` 仍写"推荐 Claude Code 当总控(Showrunner)"、`:7` 仍把 `claude -p` 当主通道——与当前"Codex 主控、Claude 冻结"直接矛盾,新接入 agent 先读 README 会被带偏。
- `cast.md:83` 自称依据 `ad-2026-06-13-04`,但 `docs/author-directives.md` 里**只有两条 06-12 记录**,没有任何 `ad-2026-06-13-*` 条目。06-13 那批最关键的裁决(冻 Claude、Codex 主控、Kiro 架构师、Cursor 总编)目前只活在 `relay/HANDOFF.md` 的散文里——而 author-directives 的设定职责恰恰是"每条指令如实转录"。这是真实溯源断点。
- 章纲 `story/chapters/ch-0001.md:15` 与正文存在设定漂移(详见第二节)。

**4. r2 闸门是"增量复审"而非全量。** `reviews/ch-0001.editorial.r2.md:6` 明说只核销地名一项。所以"双闸 pass(r2)"听起来比实际审得更狠——完整判断发生在 r1,r2 只确认一处修补,几条 craft 类 MINOR 被以"可改可留"放行。回退仅 1/4,既可能是 brief/包够好,也可能是 craft 维度闸门偏松。

## 二、`drafts/ch-0001.md` 评审

**总评:8/10 —— 一章合格、可发布的网文,而且难得地"克制得对"。** 高度忠实于 brief,POV 与信息隔离堪称教科书级,人物声音可辨,超自然异象火候压得准。问题集中在一个**创作路线判断**和几处可打磨的 MINOR。

### 做得好的(带行号)

- **危机先行,无慢热**:开篇即清册当众点名"至贱命格"+索砚(`:3-19`),黄金第一章该有的起手。文白相间稳(毡刷/墨碟/验状/乙库/经手簿),无现代词。
- **读者恰好"比沈砚多一层",不多不少**:章尾短切柯九——"脸上还挂着笑,眼底的光却已经收了"(`:65`)、"这个少年,有意思了"(`:252`)制造信息差;沈砚只能察觉"句句往旧砚上引"(`:130`)却无从破解。**全文零 `must_not_know` 泄漏**:命格被改判、碑灵之锚、封局内幕一字未碰。这是整套系统的核心承诺在终稿里真正兑现。
- **声音差异在对白层立得住**:沈砚清峻认理("凡事,得占个理"`:25`、"不急,一步一步来"`:233`)对柯九的和气藏钩("哟…说得好哇"`:35`、"敢问,您之前是何时、何处熟悉这旧契房的?"`:86`)。
- **异象火候精准**:旧砚微温"从里往外透"(`:179-181`)、纸角无风自卷(`:189`)、碑林"像有人翻了一页巨石的书"(`:193`)——全部不解释、不显字,严格命中 `sc-0001-03` brief 的"异象必须极轻"。
- **连续性修补确实落地**:验状"碑亭镇"(`:144`)、旧砚始终"裹在怀里"而把拓纸"挪到桌子的另一头"(`:211`)、夜里呼应"三日复核"(`:233`)。

### 需要定夺 / 打磨的

**[最重要 · 创作路线,非缺陷] 黄金五章 vs 琅琊榜 的内在张力,本章倒向了琅琊榜。** 第一章的"胜利"是一场**程序性、防御性**小胜(靠熟悉旧契逼退镇差),结尾沈砚仍是被压的凡俗、没有任何力量爽点。对琅琊榜式权谋读者极对味;但对典型修真网文"黄金前五章"留存曲线(期待更快的打脸/获得/升级多巴胺)的读者,一章读完"主角赢了一场官僚拉锯 + 捡到一个看不懂的痕迹"可能偏慢热。这不是 bug,是 `docs/author-directives.md` 两个范本(琅琊榜 vs 网文黄金五章)之间须由作者拍板的取舍。**建议明确记一笔目标读者**;若偏向网文留存,考虑前置更尖的钩子或压缩中段。

**[MINOR] 旧契房程序段密度偏高(`:78-148`)。** 取胜机制是甲乙丙库/经手簿/底册的"翻卷"深潜——总编 r2 也点了同条。竞争力来自"懂规矩",但对没耐心的读者是劝退点;可把逐卷翻检过程压缩一两拍提速。

**[MINOR] 沈砚对旧契房的熟稔仍略巧。** 已补"跑腿听熟的"(`:82`)和七年经手解释(`:88`),比 r1 好;若仍觉巧,可再补半句亲历细节。

**[MINOR] 柯九章尾"这个少年,有意思了"(`:252`)略俗。** 偷懒的反派定场句(总编也标了);换成更贴他"掂量算计"voice 的句子会更高级。

**[文档级,不是正文缺陷] 章纲与正文漂移,该改章纲不是改正文。** `story/chapters/ch-0001.md:15` 大纲称旧砚"在争执中出现一瞬难以解释的微温"(白天);但 `sc-0001-01` brief 明确禁止白天异象("不得…主动回应碑林"),`sc-0001-03` brief 把微温整体挪到夜里。**正文正确地跟了 brief**(微温只在夜里 `:179`)——所以要修的是那条过时的章纲大纲句,让它与已实现的正文/场景清单一致。连续性闸门没抓到它,因为它是"章纲 vs 正文"的内部漂移,不在事实源比对清单内。

## 三、如果只做三件事

1. **拍板并写死目标读者**(黄金五章留存 vs 琅琊榜权谋),据此决定 ch-0001 是否需要更尖的前置钩 / 压缩旧契房——影响全书的最高价值决定。
2. **堵住 stand-in 侵蚀**:为"DeepSeek 是 maker"的拍指定一个**非 DeepSeek 的默认 stand-in**,并在开演前做额度预检——否则额度/容量一抖,声音差异就向 DeepSeek 坍缩,正是 v1 的病。
3. **文档卫生**:更新 `README.md`(去掉 Claude 总控)、把 `ad-2026-06-13-*` 补进 `docs/author-directives.md`、修 `story/chapters/ch-0001.md:15` 的白天微温句。

**一句话总结**:系统设计与闸门实效是 9 分级的;正文是扎实的 8 分;真正的风险不在质量,而在"能不能在不被 stand-in 坍缩、不被协调成本拖垮的前提下,把这套高纪律流程坚持到 80,000 字"。
